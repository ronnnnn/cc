既存の CLAUDE.md ファイルを精査し、最適化する。

## 対象ファイル

- 引数が指定された場合: `$1/CLAUDE.md` または `$1`
- 引数がない場合: 現在のディレクトリの `CLAUDE.md`

## 精査プロセス

1. **現在の内容を読み込み**

2. **必須セクション確認**
   以下が含まれているか確認し、不足があれば追加:
   - 日本語使用時のスタイリング
   - コード参照 (LSP 使用)
   - 技術調査優先順位

3. **冗長性検出**
   以下に該当する内容を特定:
   - Claude が既知の一般的なベストプラクティス
   - 標準的なコーディング規約の説明
   - 言語・フレームワークの基本的な使い方
   - 繰り返し・重複する内容
   - 使用されていない古い情報

4. **最適化提案**
   削除・統合すべき内容をリストアップ:
   - 各項目について削除理由を説明
   - 統合可能な項目を提案

5. **ユーザー確認**
   変更内容を表示し、承認を得てから適用

## 最適化基準

### 削除対象

- IDE やエディタの使い方 → Claude は知っている
- 一般的な Git ワークフロー → Claude は知っている
- 言語の基本構文 → Claude は知っている
- フレームワークの標準的な使い方 → Claude は知っている
- 重複するコマンド説明

### 保持対象

- プロジェクト固有のワークフロー
- カスタムスクリプトやツールの使い方
- ドメイン固有の規約
- チーム固有のルール
- アーキテクチャ決定事項

## 目標

- 文字数: 500-1,500 words (最大 3,000 words)
- 構造: 明確なセクション分け
- 内容: プロジェクト固有の情報に絞る
