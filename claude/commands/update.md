# CLAUDE.md 更新ワークフロー

既存の CLAUDE.md ファイルを精査し、最適化する。

## 重要な原則

1. **冗長コンテンツを削除** - Claude が既知の一般的なベストプラクティスは含めない
2. **必須セクションを確認** - 日本語スタイリング、コード参照、技術調査優先順位
3. **肥大化を防止** - 目標 500-1,500 words、最大 3,000 words
4. **ユーザー承認を得てから適用**

## 作業開始前の準備

**必須:** 作業開始前に TodoWrite ツールで以下のステップを TODO に登録する:

```
TodoWrite([
  { content: "CLAUDE.md の読み込み", status: "pending", activeForm: "CLAUDE.md を読み込み中" },
  { content: "必須セクションの確認", status: "pending", activeForm: "必須セクションを確認中" },
  { content: "冗長性の検出", status: "pending", activeForm: "冗長性を検出中" },
  { content: "最適化提案の作成", status: "pending", activeForm: "最適化提案を作成中" },
  { content: "ユーザー承認", status: "pending", activeForm: "ユーザー承認を確認中" },
  { content: "変更の適用", status: "pending", activeForm: "変更を適用中" },
  { content: "完了報告", status: "pending", activeForm: "完了報告を作成中" }
])
```

各ステップの開始時に `in_progress` に、完了時に `completed` に更新する。

## 実行手順

### 1. 対象ファイルの決定

- 引数が指定された場合: `$1/CLAUDE.md` または `$1`
- 引数がない場合: 現在のディレクトリの `CLAUDE.md`

### 2. CLAUDE.md の読み込み

```bash
# ファイルの存在確認
ls -la CLAUDE.md 2>/dev/null
```

Read ツールで内容を読み込み、文字数を計測。

### 3. 必須セクションの確認

以下が含まれているか確認し、不足があれば追加提案:

- 日本語使用時のスタイリング
- コード参照 (LSP 使用)
- 技術調査優先順位

### 4. 冗長性の検出

以下に該当する内容を特定:

| カテゴリ                             | 判断基準         |
| ------------------------------------ | ---------------- |
| 一般的なベストプラクティス           | Claude は既知    |
| 標準的なコーディング規約             | Claude は既知    |
| 言語・フレームワークの基本的な使い方 | Claude は既知    |
| IDE やエディタの使い方               | Claude は既知    |
| 一般的な Git ワークフロー            | Claude は既知    |
| 繰り返し・重複する内容               | 1 箇所に統合可能 |
| 使用されていない古い情報             | 削除対象         |

### 5. 最適化提案の作成

削除・統合すべき内容をリストアップ:

- 各項目について削除理由を説明
- 統合可能な項目を提案
- 追加すべき必須セクションを提示

### 6. ユーザー承認

変更内容を表示し、AskUserQuestion で承認を得る:

```
## 最適化提案

### 削除対象
- [項目1]: [理由]
- [項目2]: [理由]

### 追加対象
- [必須セクション]: [不足している場合]

### 変更後の文字数
- 現在: X words
- 変更後: Y words (Z% 削減)

この変更を適用しますか？
```

### 7. 変更の適用

承認後、Edit ツールで変更を適用。

### 8. 完了報告

```
## CLAUDE.md 更新完了

- **ファイル:** <path>
- **変更前:** X words
- **変更後:** Y words
- **削減率:** Z%
```

## 保持対象

以下は削除しない:

- プロジェクト固有のワークフロー
- カスタムスクリプトやツールの使い方
- ドメイン固有の規約
- チーム固有のルール
- アーキテクチャ決定事項

## エラーハンドリング

### CLAUDE.md が存在しない場合

```
CLAUDE.md が見つかりません。/claude:init で新規作成してください。
```

### 変更が不要な場合

```
CLAUDE.md は既に最適化されています。変更は不要です。
```
