# 結果報告テンプレート

全タスクの結果を集約して以下の形式でユーザーに報告する。

## テンプレート

```markdown
## 並列タスク実行結果

**実行方式:** 並列 subagent / Agent Teams
**タスク数:** N 件
**成功:** X 件 / **失敗:** Y 件

---

### タスク 1: <タスク名> ✅

**結果:**
<結果の要約>

**変更ファイル:** (あれば)

- path/to/file1 (+10, -5)

---

### タスク 2: <タスク名> ✅ [Codex 委譲]

**Codex スレッド ID:** <スレッド ID>
**設計変更:** あり / なし
**フォールバック実行:** なし / あり (理由)

**結果:**
<結果の要約>

**変更ファイル:**

- path/to/file1 (+20, -3)
- path/to/file2 (+15, -0)

**テスト結果:**

- テストファイル: path/to/test (+30, -0)
- 結果: 全パス

**設計変更の内容:** (あれば)
<Codex が設計を変更した理由と内容>

---

### タスク 3: <タスク名> ❌

**エラー:**
<失敗の原因>

**推奨対応:**
<手動対応が必要な場合の指示>
```

## Codex 委譲タスクの報告項目

通常のタスク報告に加え、Codex 委譲タスクでは以下を含める:

- **[Codex 委譲]** ラベル - タスク名の後に付与
- **Codex スレッド ID** - 後続の追加指示に使用可能
- **設計変更の有無** - Codex が設計を変更したかどうか
- **設計変更の内容** - 変更があった場合、何をなぜ変更したか
- **テスト結果** - テストファイルのパスとパス/失敗件数
- **フォールバック実行** - Codex が利用不可で subagent が直接実装した場合に明記

## 依存関係がある場合の実行順序テンプレート

```markdown
## 実行順序

**フェーズ 1 (並列):** タスク 1, タスク 3
**フェーズ 2 (タスク 1 完了後):** タスク 2
**フェーズ 3 (全完了後):** タスク 4
```
